Frequently Asked Questions

Or questions we though were going to be asked so we just answered them in advance.

Q: What do you use to code Ralsei?
A: I use Node.js, a scripting language based on JavaScript which has proven itself time and time again to be an extremely fast and effective language.

Q: Why do you care so much for latency or restart times?
A: I want to keep the bot as fast as possible so that users won't be waiting on the bot to respond. 20 ms still means it takes 0.02s to reach the user. I want that to be as low as possible.

Q: Why do you refer to yourself as "we" on posts?
A: I don't want to grab all the credit for the bot because I didn't make it myself. I had lots of people on forum sites that provided code and tutorials, and I had all of you who made great suggestions.

Q: Why does the database disconnect often?
A: There seems to be an issue regarding stability when connecting to the database. While Ralsei continuously scans for issues, the solution may take a while to process. Sometimes we have to restart our connection, and that can take a while. Nonetheless, if your command didn't work once, try again. It could be fixed then.

Q: Why are there so many limits?
A: To streamline the release for 1.1, we need to gauge the usage of bot resources. This includes having to limit the size of each server configuration so that we don't overload the server and cause a disturbance for everyone.
We will loosen the limit on responses once we are using a database with sufficient space.
